Tesla Fleet API Guide

Connect Tesla Fleet in 10 Minutes

Follow this checklist to generate an access token, connect your vehicle, and verify lock chime compatibility safely.

Estimated time: 10-15 min Need: Tesla account + browser

Step-by-Step

  1. Open Tesla Developer Portal

    Go to developer.tesla.com, sign in with your Tesla account, then open Fleet API settings.

  2. Create an OAuth App

    Set app name, allowed redirect URI, and generate client credentials for personal use.

  3. Request Vehicle Scopes

    Enable required scopes like vehicle_device_data and vehicle_cmds. Start with minimum scope.

  4. Issue Access Token

    Run your OAuth flow and copy the access token. Keep it private and do not upload it to Git.

  5. Paste Token in App

    In Tesla Lock Sound Creator, paste token into Tesla Fleet access token input and click Connect Tesla.

  6. Select Vehicle and Check

    Choose your vehicle from the list and click Check Compatibility to validate readiness.

  7. Rotate Token Regularly

    Revoke old tokens and rotate credentials periodically for account safety.

Security Checklist

  • Never share access tokens in screenshots or support chats.
  • Store tokens in secure vaults, not plain text files.
  • Use shortest practical token expiration.
  • Immediately revoke token if leakage is suspected.

Troubleshooting

401 Unauthorized: Token expired or invalid. Regenerate token and reconnect.

403 Forbidden: Scope is insufficient. Reconfigure OAuth scopes in Tesla Developer portal.

No vehicles returned: Check if account has an active vehicle and correct region/base URL.
